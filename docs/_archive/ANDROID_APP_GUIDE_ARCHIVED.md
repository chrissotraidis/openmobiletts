# Android App Guide

## Overview

Open Mobile TTS runs on Android via **Capacitor** — the same SvelteKit web app wrapped in a native Android shell. The Android app loads the UI in a WebView and connects to your Python TTS server over your local WiFi network.

This means: one codebase. Change the web app, rebuild, same changes on Android.

## How It Works

```
┌──────────────────────┐         ┌──────────────────────┐
│  Android Phone       │  WiFi   │  Your Computer       │
│                      │         │                      │
│  Capacitor WebView   │◄───────►│  python run.py       │
│  (SvelteKit SPA)     │  HTTP   │  (FastAPI + Kokoro)  │
│                      │         │  port 8000           │
│  Settings → Server   │         │                      │
│  URL configured here │         │                      │
└──────────────────────┘         └──────────────────────┘
```

The server runs on your computer. The Android app connects to it over your local network.

## Prerequisites

- **Android Studio** (latest stable)
- **JDK 17** (bundled with Android Studio)
- **Android device** or emulator (API 22+, Android 5.1+)
- **Computer running the server** (`python run.py`)
- Both devices on the **same WiFi network**

## Setup

### 1. Start the Server

On your computer:

```bash
python run.py
# Server starts at http://0.0.0.0:8000
```

Note your computer's local IP address:
- **macOS**: System Settings > Wi-Fi > Details > IP Address
- **Linux**: `hostname -I`
- **Windows**: `ipconfig` (look for IPv4 Address)

### 2. Open in Android Studio

```bash
# From the project root
cd client/android
```

Open this directory in Android Studio. It will:
1. Detect the Gradle project
2. Sync dependencies (first time takes a few minutes)
3. Index the project

### 3. Build and Run

1. Connect your Android device via USB (or start an emulator)
2. Click **Run** (green play button) in Android Studio
3. Select your device
4. The app installs and launches

### 4. Configure Server Connection

In the Android app:
1. Go to **Settings** tab (bottom nav)
2. Find the **Server Connection** card
3. Enter your server URL: `http://<your-computer-ip>:8000`
4. Tap **Test Connection**
5. You should see "Connected (v0.2.0)"

Now use the app normally — Generate, History, and Settings all work.

## Building for Release

### Build the Web Assets First

```bash
cd client
npm run build:android
```

This runs `vite build` then `npx cap sync android`, which copies the latest web build into the Android project.

### Build APK in Android Studio

1. Open `client/android/` in Android Studio
2. Build > Build Bundle(s) / APK(s) > Build APK(s)
3. The APK is at `client/android/app/build/outputs/apk/debug/app-debug.apk`

### Install APK on Device

```bash
adb install client/android/app/build/outputs/apk/debug/app-debug.apk
```

## Development Workflow

When you change the web app:

```bash
cd client
npm run build:android    # Rebuild + sync to Android
```

Then run again from Android Studio. The Android project always uses the latest built web assets.

## Troubleshooting

### "Could not reach server" on Test Connection

- Verify both devices are on the same WiFi network
- Check the server is running: `curl http://<your-ip>:8000/api/health`
- Try `http://` not `https://` (the server uses HTTP)
- Check your computer's firewall allows connections on port 8000

### Gradle sync fails in Android Studio

- Ensure Android Studio is using JDK 17 (Settings > Build > Gradle > JDK)
- Try File > Invalidate Caches and Restart

### App shows white screen

- Make sure you've run `npm run build:android` at least once
- Check that `client/android/app/src/main/assets/public/` contains the built web files

### Audio doesn't play

- Verify the server connection is working (Test Connection in Settings)
- Check the server logs for errors
- Try generating a short text first

## Project Structure

```
client/android/
├── app/
│   ├── src/main/
│   │   ├── AndroidManifest.xml      # Permissions, cleartext traffic
│   │   ├── java/.../MainActivity.java   # Capacitor bridge
│   │   ├── assets/public/           # Built web app (generated)
│   │   └── res/                     # Icons, splash screens, strings
│   └── build.gradle                 # App-level build config
├── build.gradle                     # Project-level build config
├── variables.gradle                 # SDK versions, dependency versions
├── capacitor.settings.gradle        # Capacitor module references
└── gradle/                          # Gradle wrapper
```

Files managed by Capacitor (regenerated by `cap sync`):
- `app/src/main/assets/public/` — built web assets
- `app/src/main/assets/capacitor.config.json` — runtime config
- `capacitor-cordova-android-plugins/` — plugin bridge

## Key Configuration

### AndroidManifest.xml

- `android:usesCleartextTraffic="true"` — allows HTTP connections to the local server
- `android.permission.INTERNET` — required for network access

### capacitor.config.ts

```typescript
{
  appId: 'com.openmobiletts.app',
  appName: 'Open Mobile TTS',
  webDir: 'build',
  android: {
    allowMixedContent: true,  // HTTPS WebView making HTTP API calls
  },
}
```
